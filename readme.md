# Симуляция муравьиной колонии

Консольная симуляция жизненного цикла муравьиной колонии с учетом ресурсов, старения, болезней, атак и развития потомства.  
Проект реализован в объектно-ориентированном стиле и моделирует сложные взаимодействия внутри колонии.

---

## Общая логика программы

Симуляция выполняется **пошагово по дням**. Каждый день колония проходит следующие этапы:

1. Проверка случайных событий (атаки)
2. Сбор пищи рабочими муравьями
3. Кормление королевы, личинок и взрослых муравьев
4. Работа королевы (откладка яиц)
5. Рост личинок и переход в стадию куколок
6. Превращение куколок во взрослых муравьев
7. Старение всех муравьев и проверка условий смерти
8. Сбор и вывод статистики

Симуляция завершается:
- по достижении заданного числа дней  
- либо при гибели колонии (смерть королевы или отсутствие живых муравьев)

## Структура проекта

```text
ants/
├── base.py          # базовый класс Ant
├── worker.py        # рабочий муравей, наследует Ant
├── soldier.py       # солдат, наследует Ant
├── queen.py         # королева, наследует Ant
├── larva.py         # личинка и куколка, наследует Ant

core/
├── colony.py        # класс AntColony, управляющий симуляцией
├── config.py        # SimulationConfig, конфигурация симуляции, правится руками
├── ant_state.py     # перечисление состояний муравьев, косячное lavra и pupa перенести в ant_stage, добавить state.Молодой
├── events.py        # базовый класс событий
├── attack_event.py  # событие атаки, наследует ColonyEvent

main.py              # точка входа в программу
```

## Результат работы

По завершении симуляции пользователю выводится подробный отчет о состоянии колонии:

- состояние королевы (жива / погибла, возраст, здоровье)
- текущая численность колонии с разбивкой по ролям
- агрегированная статистика смертности
- информация о произошедших событиях (атаки, потери, критические ситуации)
- возможность сохранить итоговый отчет в текстовый файл для последующего анализа

## Возможности расширения

Архитектура проекта позволяет легко расширять и усложнять симуляцию:

- добавление новых типов событий (эпидемии, миграции, погодные условия), основываясь на events
- введение новых ролей муравьев (разведчики, строители, лекари), основываясь на ant
- более сложная экономика ресурсов (разные типы пищи, хранение, порча)
- сохранение и загрузка состояния симуляции для продолжения или анализа
